Les API

  1. Présentation
  2. Fonctionnement

Présentation

Le terme API signifie Application Programming Interface. Il s'agit d'une interface qui permet à plusieurs applications de communiquer entre elles. L'API d'une application permet un accès aux données de celle-ci. L'accès aux données se fait de façon structuré. Une application cliente va utilisée l'API d'une autre application afin d'accéder directement aux données sans changer d'application. Voici une courte vidéo explicative.

Fonctionnement

Pour faire fonctionner une API, il faut un client et un serveur. Le serveur met à disposition et éxecute le programme de l'API. Le serveur attend qu'on lui envoie une demande sous forme de requête, executera l'API et renverra des données de façon structurées. Le client enverra une requête et recevra les données de façon brute (sans interface utilisateur) qu'il pourra alors utiliser. Ceci est transparent pour l'utilisateur qui lui restera sur l'application cliente. Aujourd'hui de nombreuses applications utilisent des API, ceci évite de redévelopper un système qui fonctionnne déjà et qui est mis à disposition. Par exemple, l'application Uber Eats va utiliser l'API de google afin de récupérer la géolocalisation de l'utilisateur. Ceci est totalement transparent pour l'utilisateur qui restera dans son application Uber Eats. Pour pouvoir être utilisée sans difficulté, les API ont une architecture de type REST ou SOAP. Aujourd'hui, la majorité ont une architecture de type REST. Chaque type d'architecture impose des contraintes.
